# CloudLaunch

[![MIT License](https://img.shields.io/badge/License-MIT-green.svg)](LICENSE)

**CloudLaunch** は、PC ゲームのセーブデータを S3 互換のクラウドストレージへ差分同期し、複数の PC で進行状況を持ち運べるようにするデスクトップアプリです。プレイ時間の自動計測、ルート単位の進行管理、Markdown プレイメモ、スクリーンショット、まとめてバックアップなど、ひとつのゲームライブラリで完結する周辺機能も備えています。

> [!NOTE]
> **メイン実装は Go / Wails 版(本リポジトリ)です。**
> 旧 [Electron 版](https://github.com/fuyu28/CloudLaunch) は参照用に残していますが、機能追加と修正は本リポジトリで継続します。

## ✨ 主な機能 (Features)

### 🎮 ゲームライブラリ

- **ゲーム登録**: 実行ファイル・セーブフォルダ・カバー画像をひもづけて管理
- **ErogameScape インポート**: URL またはタイトル検索でタイトル名・ブランド名・カバー画像を自動取得
- **プレイ状況**: `未プレイ` / `プレイ中` / `プレイ済み` を手動切替、クリア日付の記録でステータスを自動上書き
- **ルート管理**: 1 つのゲームに複数のルート(攻略 / 周回)を持たせ、セッションをルート単位で記録

### ⏱️ プレイトラッキング

- **自動セッション記録**: 実行中プロセスを検出してプレイ時間を計測(Windows のみ)
- **手動セッション**: プロセス検出が効かない環境では Start / Stop ボタンで記録
- **統計**: 総プレイ時間・直近 7 日間・セッション履歴

### ☁️ セーブデータ同期(content-addressing 方式)

- **差分同期**: ファイル内容のハッシュ単位で blob を保存し、変更があった分だけアップロード / ダウンロード
- **整合性検証**: 受信した blob はコンテンツハッシュで検証
- **コンフリクト検出**: 別 PC で進めた変更とローカルが衝突した場合は確認ダイアログで「ローカル採用 / リモート採用」を選択
- **untracked 削除確認**: Pull 時に同期対象外のローカルファイルが消えそうな場合は事前確認
- **ゲーム単位のクラウドデータ閲覧 / 削除**: クラウドデータ管理ページから内容確認・ゲーム単位の削除が可能
- **オフラインモード**: 同期は無効化、ローカル機能のみ稼働

### 📝 プレイメモ

- **Markdown エディタ**: リアルタイムプレビュー、画像挿入
- **ゲーム別整理**: 1 ゲームに複数メモ
- **クラウド同期**: メモ単位でアップロード / ダウンロード

### 🖼️ スクリーンショット(Windows)

- **ホットキー撮影**: ゲームウィンドウをカスタムキーで撮影
- **ゲーム別保存**: 撮影画像をゲーム別ディレクトリに整理

### 💾 データ管理

- **エクスポート**: ゲーム情報・セッション・メモを JSON で書き出し
- **フルバックアップ**: DB と関連ファイルをまとめてアーカイブ
- **復元**: バックアップから一括復元

### 🎨 UI

- **独自テーマ**: 読みやすさを軸に調整した `cloudlaunch` テーマ(明るめ)
- **OS 別タイトルバー**: Windows は独自スタイル、macOS / Linux はネイティブを尊重

## 🚀 使い方 (How to Use)

### インストール(推奨)

GitHub Releases から最新の `CloudLaunch-Setup-x.y.z.exe`(Windows)をダウンロードして実行してください。

### 初回セットアップ

1. アプリを起動
2. 設定画面 → 「クラウド」タブで S3 互換ストレージの接続情報を入力
3. ゲームを登録(実行ファイル / セーブフォルダ / カバー画像)
4. 詳細画面の「クラウドデータ管理」から `アップロード` / `ダウンロード` を実行

### 動作確認済みの S3 互換ストレージ

- **AWS S3**
- **Cloudflare R2**
- **MinIO**

(その他 S3 互換 API に準拠したストレージなら基本的に動作するはずです)

### オフラインモード

設定画面で有効化できます。クラウド系の操作はバックエンドごと無効化され、ローカルのゲーム管理 / メモ / スクリーンショットは引き続き利用できます。

## 🔐 セキュリティとプライバシー

- **認証情報の保存**: S3 / R2 の Access Key / Secret Key は OS のキーチェーン(Windows: Credential Manager)に保存します。プレーンなファイルや DB には書き出しません。
- **外部送信**: 認証情報はユーザーが指定した S3 互換エンドポイントへの認証目的でのみ使用されます。**開発者を含む第三者には一切送信されません。**
- **テレメトリ**: 利用統計やクラッシュレポートを外部に送信する仕組みはありません。

## 🛠️ 開発者向け (for Developers)

### 必要要件

- Go 1.25 以上(`go.mod` 準拠)
- [Wails v2](https://wails.io/) (`v2.11.0` 以上を想定)
- [Bun](https://bun.sh/)
- Windows 向けインストーラビルドのみ NSIS(Wails が自動で呼ぶ)

### セットアップと開発起動

```bash
# クローン
git clone https://github.com/fuyu28/CloudLaunch_Go.git
cd CloudLaunch_Go

# フロントエンド依存を入れる
cd frontend && bun install && cd ..

# 開発モードで起動
wails dev
```

### よく使うコマンド

```bash
# Go テスト
go test ./...

# フロントエンドテスト
cd frontend && bun run test

# lint + format(タスク完了時に実行)
./scripts/run-all-lint-format.sh

# 本番ビルド(プラットフォーム別バイナリ)
wails build
```

### 技術スタック

**フロントエンド**

- React + TypeScript(Vite ビルド)
- Tailwind CSS + DaisyUI(独自テーマ `cloudlaunch`)
- 状態管理: Jotai
- ルーティング: React Router

**バックエンド**

- 言語: Go
- フレームワーク: Wails v2
- データベース: SQLite(生 SQL を `internal/infrastructure/db/repository.go` に直書き、マイグレーションは連番 SQL ファイル)
- クラウド接続: AWS SDK for Go v2(S3 互換)
- 認証情報: Windows Credential Manager(macOS / Linux は未対応)

**設計**

- Clean Architecture 4 層構成(`domain` / `infrastructure` / `services` / `app`)
- セーブデータ同期: content-addressing 方式(commit / tree / blob を hash key で保存)

### プロジェクト構造

```
CloudLaunch_Go/
├── internal/
│   ├── domain/           # モデル・型(外部依存なし)
│   ├── infrastructure/   # DB / S3 / 認証情報の実装
│   ├── services/         # ユースケース
│   └── app/              # Wails アダプター層
├── frontend/             # Vite + React UI
├── build/                # Wails ビルド用アセット(アイコン / NSIS テンプレ)
├── docs/                 # 設計メモ・リファクタ計画
├── scripts/              # 開発支援スクリプト
├── main.go               # Wails エントリポイント
└── wails.json            # Wails 設定
```
